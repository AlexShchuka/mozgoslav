# api-rest-to-graphql

## decision
- One `/graphql` endpoint replaces the per-domain REST surface. Same endpoint serves HTTP queries/mutations and `graphql-ws` subscriptions over WebSocket.
- Three REST endpoints survive on technical grounds; everything else is removed.
- Loopback-only posture preserved. No auth introduced. No outbound traffic added.

## stack
- server — HotChocolate 14.x, code-first, Relay-compliant. `AddGlobalObjectIdentification`, `AddFiltering`, `AddSorting`, `AddProjections`, `AddInMemorySubscriptions`. Strict validation on. Introspection gated to dev.
- pagination — cursor (`[UsePaging]`); default page 25, max 200, total-count on, boundaries optional.
- batching — DataLoader (`[DataLoader]` attribute) for every parent→children surface; batch methods on the repository, not in the loader body.
- projections — `[UseProjection]` on every EF-backed query; `[UseFiltering]` + `[UseSorting]` on list queries powering tables.
- mutations — errors-as-data: payload `{ entity, errors: [IUserError!]! }`. `IUserError` interface with concretes `ValidationError`, `NotFoundError`, `ConflictError`, `UnavailableError`. `throw` reserved for infrastructure defects.
- subscriptions — `HotChocolate.Subscriptions.InMemory`. Bridge `Channel<T>` notifiers to `ITopicEventSender`/`ITopicEventReceiver` via `IHostedService` adapters.
- auth — none. No `[Authorize]`. No JWT.
- validation — no FluentValidation. Validate at resolver entry; return `UserError` on failure.
- schema snapshot — `Snapshooter.MSTest` test verifies the live SDL against a checked-in `.snap`. Root-level SDL artefact is regenerated by a console command.
- client — `graphql-request` 7.x (HTTP) + `graphql-ws` 5.x (subscriptions), lazy WebSocket init.
- codegen — `@graphql-codegen/cli` `client-preset`. Generated `TypedDocumentNode`s committed alongside code; regenerated only by an explicit `codegen` script.
- operations — one `.graphql` file per domain. Document names prefixed `Query…` / `Mutation…` / `Subscription…`.
- CI drift guards — schema drift step (export + `git diff --exit-code`) and codegen drift step (`npm run codegen` + `git diff --exit-code`).

## REST survivors
- `POST /_internal/devices/changed` — inbound from the Swift helper over the loopback bridge.
- `POST /_internal/hotkey/event` — inbound from the Swift helper.
- `POST /api/dictation/{sessionId}/push` (raw octet-stream) and `POST /api/dictation/push/{sessionId}` (JSON variant) — binary PCM streaming from the renderer; gql is the wrong shape for binary frames.
- All three live in a single `InternalEndpoints` module. Everything else REST is removed; `AddControllers` / `MapControllers` is gone.

## frontend
- Sagas consume gql via a typed `gqlRequest(Document, vars)` helper and an `eventChannel`-wrapped `getGraphqlWsClient().subscribe` for subscriptions.
- The single surviving REST endpoint is reached via a thin `dictationPush` helper using `fetch`.
- Per-domain axios clients, the `ApiFactory`, `BaseApi`, and `axios` itself are removed except where the obsidian saga migration is pending (tracked separately).

## domain coverage
Mapped per the migration phases:
- health + meta, settings (with the `IUserError` model), profiles (first `[Node]`), models + `modelDownloadProgress` subscription, recordings (paged + DataLoaders), notes (paged + export), jobs + `jobProgress`, dictation subscriptions (events, audio-device-changed, hotkey), obsidian + meetily + backup, rag (query + reindex), sync + `syncEvents`.
- Phase 12 cutover removed REST endpoints and per-domain axios clients. Phase 12c added the missing CRUD ops the earlier phases skipped: `deleteRecording`, `deleteNote`, `createNote`, `ragStatus`, and `recording.notes` via DataLoader.
- The logs feature was removed entirely after migration — Serilog file output is the single source of truth, no in-app viewer.

## what it does NOT do
- No public exposure: endpoint stays loopback-only.
- No auth surface.
- No outbound traffic beyond what already existed.
- No backwards-compat REST routes left as a safety net.
- No multipart file upload via gql; uploads stay path-based (`UploadRecordingsInput.FilePaths`).
- No federation, no subscriptions over SSE, no `@defer`/`@stream` directives.

## rejected
| alt | reason |
|---|---|
| schema-first SDL + server codegen | code-first sufficient for a single-client project; snapshot test guards drift. |
| Redis / MQTT pub-sub for subscriptions | loopback desktop; in-memory pub-sub is enough. |
| Apollo Client | redundant cache layer — Redux owns state; `graphql-request` keeps the client surface minimal. |
| JWT / session auth | loopback-only posture; out of scope per the privacy ADR. |
| FluentValidation | resolver-entry validation covers the small surface; not worth a new package. |
| Keep REST alongside gql indefinitely | duplicated transport, DTO drift, doubled test surface. |
| Migrate DTOs before resolvers | domain churn with no visible value until the surface flips. |
| First domain = dictation | heavy subscription surface; chose health + meta first to warm up the patterns. |
| Base64-encode binary PCM through a gql mutation | 33% size inflation, JSON overhead, no streaming; REST survivor is the right shape. |
