# api-rest-to-graphql-checklist

## goal
- One gql surface replaces every REST endpoint and every SSE stream.
- Loopback posture preserved. No auth introduced. No telemetry added.
- Application use cases unchanged; migration is a transport swap.

## non-goals
- Changes to domain, persistence, or use-case layers.
- Public exposure of the gql endpoint.
- Authentication surface.
- Outbound traffic beyond what already exists.
- Edits to anything already under `.archive/`.

## fixed stack (do not relitigate)
- server — HotChocolate 14.x via central package management, floating minor.
- approach — code-first with descriptor classes; schema emitted from types.
- relay — `AddGlobalObjectIdentification`; every entity has `[Node]` + `NodeResolver`.
- pagination — cursor (`[UsePaging]`), include-total-count on, default 25, max 200, boundaries not required.
- batching — DataLoader (`[DataLoader]` attribute style) mandatory on every parent→children surface; batch method lives on the repository, not inside the loader.
- projections — `[UseProjection]` on every EF-backed query; `[UseFiltering]` + `[UseSorting]` on list queries that power tables.
- mutations — errors as data. Payload shape `{ entity: T?, errors: [IUserError!]! }`. `throw` reserved for infrastructure defects.
- subscriptions — `HotChocolate.Subscriptions.InMemory`; transport `graphql-ws` over WebSocket on the same gql endpoint.
- auth — none. No `[Authorize]`. No JWT middleware.
- validation — no FluentValidation; validate at resolver entry and return a `UserError` on failure.
- schema snapshot — root `schema.graphql` regenerated by a console command; `Snapshooter.MSTest`-based test guards drift.
- client — `graphql-request` 7.x + `@graphql-codegen/cli` 5.x `client-preset` + `graphql-ws` 5.x.
- generated output — committed alongside code; regenerated only by an explicit `codegen` script.
- operations — one `.graphql` file per domain; document names prefixed `Query…` / `Mutation…` / `Subscription…`.

## error model
- interface `IUserError { code, message }`.
- concretes: `ValidationError(code, message, field)`, `NotFoundError(code, message, resourceKind, resourceId)`, `ConflictError(code, message)`, `UnavailableError(code, message)`.
- every mutation payload exposes `errors: [IUserError!]!`.
- generic handlers accept the interface; no resolver hard-codes a concrete error type.

## endpoint inventory (source of scope)
- health → `Query.health`, `Query.llmHealth`.
- meta → `Query.meta`.
- settings → `Query.settings`, `Mutation.updateSettings`.
- profiles → `Query.profiles` / `Query.profile`, `Mutation.createProfile` / `updateProfile` / `deleteProfile`.
- models → `Query.models`, `Mutation.downloadModel`, `Subscription.modelDownloadProgress`.
- recordings → `Query.recordings` (paged) / `Query.recording`, `Mutation.importRecordings` / `uploadRecordings` / `reprocessRecording` / `startRecording` / `stopRecording`.
- jobs → `Query.jobs` (paged) / `Query.activeJobs`, `Mutation.enqueueJob` / `cancelJob`, `Subscription.jobProgress` (SSE retired).
- notes → `Query.notes` (paged) / `Query.note`, `Mutation.exportNote`.
- meetily → `Mutation.importFromMeetily`.
- obsidian → `Query.obsidianDetect`, `Mutation.setupObsidian`.
- backup → `Query.backups`, `Mutation.createBackup`.
- dictation → `Query.dictationAudioCapabilities` / `…status`, `Mutation.dictationStart` / `stop` / `…`, `Subscription.dictationEvents`, `Subscription.audioDeviceChanged`, `Subscription.hotkeyEvents`.
- sync → `Query.syncStatus` / `syncHealth` / `syncPairingPayload`, `Mutation.acceptSyncDevice`, `Subscription.syncEvents` (SSE retired).
- rag → `Mutation.ragReindex`, `Query.ragQuery`.
- logs → `Query.logs`, `Query.logTail(file, lines)` — replaces the MVC logs controller.

## explicit REST survivors
- `POST /_internal/devices/changed` — inbound from the Swift helper over the Electron loopback bridge; stays REST.
- `POST /_internal/hotkey/event` — inbound from the Swift helper; symmetric surface; stays REST.
- Only the outbound SSE fan-outs for device changes and hotkey events migrate to subscriptions.

## operational rules (every phase)
- read before write — existing endpoint, repository interface, domain entity, frontend client, consuming saga.
- grep before create — no duplicate symbols; reuse existing types.
- one phase == one commit == one logical surface.
- 2-strike rule — same step failing twice halts the phase; capture context, report, wait. No silent third retry.
- autonomy — no mid-execution questions to the user; pick the best default consistent with this document and flag it `UNVERIFIED` in the phase report.
- commands — every `dotnet` invocation uses `-maxcpucount:1`.
- git — no `push`, no MR, no `main` / `master`, no `--force`, no hook-skip, no amend without explicit instruction.
- scope — no drive-by refactors; unrelated bugs go into the phase report, not into code.
- archive — never read `.archive/` as source of truth; never edited.

## phases

### phase 0 — scaffolding
- [ ] add HotChocolate AspNetCore + Data + Data.EntityFramework + Subscriptions.InMemory + GreenDonut.Selectors via central package management.
- [ ] add `Snapshooter.MSTest` package for schema snapshot tests.
- [ ] wire one gql endpoint on the loopback host for both HTTP and WebSocket.
- [ ] register empty root Query, Mutation, Subscription types; chain per-domain extension points.
- [ ] enable global object id, filtering, sorting, projections, paging options, in-memory subscriptions.
- [ ] hide exception detail; gate introspection to dev builds.
- [ ] add a schema-snapshot test project with one `Snapshooter.MSTest`-based test that calls `.MatchSnapshot()` on the live SDL.
- [ ] add a console command that prints live SDL to the root `schema.graphql`.
- [ ] frontend: install `graphql`, `graphql-request`, `graphql-ws`, `@graphql-codegen/cli`, `client-preset`.
- [ ] add codegen config consuming the root SDL + per-domain operation files.
- [ ] add `codegen` and `codegen:watch` npm scripts.
- [ ] add a shared gql client module — HTTP client eager, WebSocket client lazy.
- [ ] add saga helpers for typed query / mutation + subscription event-channel.
- [ ] extend renderer CSP to allow the loopback WebSocket path.
- [ ] exit — `{ __typename }` returns `Query`; schema snapshot committed; frontend typecheck + lint + tests green.

### phase 1..11 — per-domain
- order designed so each phase depends only on types from earlier phases:
  - [ ] 1 — health + meta.
  - [ ] 2 — settings (introduces `IUserError` interface + concretes).
  - [ ] 3 — profiles (first `[Node]` entity; template for later entities).
  - [ ] 4 — models (first subscription; establishes the `ITopicEventSender` / `ITopicEventReceiver` bridge over existing `Channel<T>` fan-outs).
  - [ ] 5 — recordings (first paged query; first `[UseFiltering]` / `[UseSorting]` + DataLoader set).
  - [ ] 6 — notes (paged; depends on the recording Node).
  - [ ] 7 — jobs + `jobProgress` subscription.
  - [ ] 8 — dictation subscriptions (events, audio-device-changed, hotkey).
  - [ ] 9 — obsidian + meetily + backup + logs (grouped).
  - [ ] 10 — rag.
  - [ ] 11 — sync + `syncEvents` subscription.
- per-phase template:
  - [ ] read the endpoint file, repository interface, domain entity, frontend client, consuming saga.
  - [ ] add schema types, queries, mutations, subscriptions under a per-domain gql namespace.
  - [ ] resolvers delegate to existing use cases; no business logic moves into the gql layer.
  - [ ] add `[Node]` + `NodeResolver` on each entity; add DataLoaders for every parent→children surface; add batch methods on repositories where missing.
  - [ ] per-domain mutation inputs + payloads; errors as data; no throws for expected failures.
  - [ ] resolver integration tests cover zero / one / paged / invalid-cursor for queries, and success + every error branch for mutations; DataLoader batch asserted called once per resolver pass.
  - [ ] add per-domain operation documents; run codegen; commit regenerated client.
  - [ ] rewire sagas to typed gql ops; do not delete the old per-domain axios client yet.
  - [ ] regenerate root SDL; verify schema snapshot; approve diff.
  - [ ] backend + frontend build + tests green on touched projects.
  - [ ] commit once per phase.

### phase 12 — cutover
- [ ] remove migrated REST endpoint files and their registration calls.
- [ ] extract the surviving internal POSTs (device-change + hotkey event) into a minimal internal-endpoint module.
- [ ] remove the logs MVC controller and the `AddControllers` / `MapControllers` wiring once logs migrated.
- [ ] delete per-domain axios clients, api factory, and api constants module.
- [ ] remove `axios` from frontend dependencies.
- [ ] grep-guard — zero remaining references to migrated REST paths in backend or frontend sources; only the two `_internal` POSTs stay.
- [ ] full backend and frontend suites green.
- [ ] regenerate SDL; confirm snapshot stable.
- [ ] move the previous rest-to-graphql backlog notes to `.archive/`.
- [ ] publish a shipped decision under the api feature capturing the final surface.

### phase 13 — CI drift guard
- [ ] add a CI step running the SDL export and failing on `git diff --exit-code` drift.
- [ ] add a CI step running `codegen` and failing if generated output is out-of-sync.
- [ ] snapshot review — scan final `schema.graphql` for leaked internal fields, missing `UserError` members, misplaced `@deprecated`.

## commit schema
- conventional commits; one commit per phase.
- subjects:
  - `feat(graphql): phase 0 — scaffold HotChocolate server + codegen client`
  - `feat(graphql): phase 1 — health + meta`
  - `feat(graphql): phase 2 — settings + UserError model`
  - `feat(graphql): phase 3 — profiles`
  - `feat(graphql): phase 4 — models + modelDownloadProgress subscription`
  - `feat(graphql): phase 5 — recordings (pagination + DataLoaders)`
  - `feat(graphql): phase 6 — notes (pagination + export mutation)`
  - `feat(graphql): phase 7 — jobs + jobProgress subscription`
  - `feat(graphql): phase 8 — dictation subscriptions`
  - `feat(graphql): phase 9 — obsidian + meetily + backup + logs`
  - `feat(graphql): phase 10 — rag (query + reindex)`
  - `feat(graphql): phase 11 — sync + syncEvents subscription`
  - `feat(graphql): phase 12 — cutover (remove REST, delete api classes)`
  - `ci(graphql): phase 13 — schema drift guard + codegen verification`
- body lists surface added / removed, test counts, DataLoader count, subscription topics, UNVERIFIED notes. No emoji. No Co-Authored-By unless requested.

## per-phase exit checklist
- [ ] backend build green; zero new warnings (`-maxcpucount:1`).
- [ ] backend tests green; new tests cover success and every error branch.
- [ ] frontend typecheck, lint, tests green.
- [ ] root SDL matches live schema.
- [ ] schema snapshot approved.
- [ ] generated gql client committed.
- [ ] no axios per-domain client deleted in phases 0..11.
- [ ] no push, no MR, no `main` touched.

## report
- phase commits append to a working report file outside the repo (sandbox path).
- per phase — number, SHA, files added / modified / deleted, test counts, DataLoader count, subscription topics added, UNVERIFIED notes.
- final summary after phase 13 — total commits, total DataLoaders, total subscriptions, total resolver tests, SDL line count.

## rejected
| alt | reason |
|---|---|
| schema-first SDL + server codegen | code-first already sufficient for a single-client project; snapshot test guards drift. |
| Redis / MQTT pub-sub for subscriptions | loopback desktop; in-memory pub-sub sufficient. |
| Apollo Client | caching layer redundant — Redux owns state; `graphql-request` + codegen keeps the client surface minimal. |
| JWT / session auth | loopback-only posture; out of scope per the privacy ADR. |
| FluentValidation | six resolvers worth of validation does not justify a new package. |
| REST alongside gql indefinitely | duplicated transport, DTO drift, doubled test surface. |
| DTO-first migration | domain churn with no visible value until the surface flips. |
| First domain = dictation | heavy subscription surface; wrong first domain. |
